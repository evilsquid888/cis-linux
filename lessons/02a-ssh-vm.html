Lesson 2a — SSH Hardening · with embedded Linux

CIS §5.2 ~15 min 4 controls ← read-only edition
Same content as Lesson 2, with a real Linux VM you can type into. Click ▶ Boot VM on the right (about 5 seconds to boot + stage). The shell starts with a deliberately weak /etc/ssh/sshd_config plus stub sshd / sudo / systemctl shims so the canonical CIS audit commands actually work end-to-end.

🎯 Why this matters

SSH is the door to the box. The default OpenSSH config on Ubuntu permits root login, password authentication, and unlimited idle sessions — every one of those is an attacker's best friend. Brute-force scanning of port 22 happens constantly; a weak password on a root account is game-over.

This lesson tightens four CIS controls: disable root SSH login, disable password auth in favor of keys, drop MaxAuthTries from 6 to 4, and add an idle-session timeout. Each is one or two lines in /etc/ssh/sshd_config.

📖 Tools used in this lesson

sshd — OpenSSH server daemon. Reads /etc/ssh/sshd_config at startup and on SIGHUP. The flag sshd -T dumps the *effective* runtime config (file values merged with compile-time defaults) — the canonical "what would the daemon actually do" view CIS auditors use. man 8 sshd · man 5 sshd_config · OpenSSH manual index

sed — stream editor. The remediate block uses sed -i 's/old/new/' to do in-place find-and-replace across the config. The ^#* in the patterns matches "zero or more leading #" so the same expression handles both already-set and commented-out directives. man 1 sed · GNU sed manual

grep — pattern search. grep -q '^Pattern' exits silently with status 0 (match) or 1 (no match), perfect for a guard before echo … >> file. man 1 grep

systemctl reload — tells systemd to send SIGHUP (or whatever the unit's ExecReload is) to the running daemon, prompting it to re-read its config without dropping connections. Distinct from restart which kills+respawns. man 1 systemctl

sudo — runs the next command as root after authenticating the invoker against /etc/sudoers. man 8 sudo

Refresher: what is sshd_config?

/etc/ssh/sshd_config configures the SSH server daemon (sshd). Edits take effect after systemctl reload ssh. The companion file /etc/ssh/ssh_config configures the SSH client — leave that alone.

🔍 Audit — find current state

Use sshd -T to see the effective running config (file + defaults merged) — this is what the daemon would actually behave like at runtime, and what CIS auditors check:

sudo sshd -T | grep -iE 'permitrootlogin|passwordauth|maxauthtries|clientalive'

You can also read the file directly:

cat /etc/ssh/sshd_config

CIS-recommended targets:

  • PermitRootLogin = no
  • PasswordAuthentication = no (only after a working SSH key is in place)
  • MaxAuthTries = 4
  • ClientAliveInterval = 300, ClientAliveCountMax = 2

🔧 Remediate

Lab-only shortcut

On a real box you'd open a second SSH session before changing these settings, to avoid locking yourself out. The VM here is local, so you can experiment freely — break it, reset it, retry.

Apply all five directives at once with sed, then append any that weren't already present:

sed -i \
  -e 's/^#*PermitRootLogin .*/PermitRootLogin no/' \
  -e 's/^#*PasswordAuthentication .*/PasswordAuthentication no/' \
  -e 's/^#*MaxAuthTries .*/MaxAuthTries 4/' \
  -e 's/^#*ClientAliveInterval .*/ClientAliveInterval 300/' \
  -e 's/^#*ClientAliveCountMax .*/ClientAliveCountMax 2/' \
  /etc/ssh/sshd_config

grep -q '^PermitRootLogin'        /etc/ssh/sshd_config || echo 'PermitRootLogin no'        >> /etc/ssh/sshd_config
grep -q '^PasswordAuthentication' /etc/ssh/sshd_config || echo 'PasswordAuthentication no' >> /etc/ssh/sshd_config
grep -q '^MaxAuthTries'           /etc/ssh/sshd_config || echo 'MaxAuthTries 4'             >> /etc/ssh/sshd_config
grep -q '^ClientAliveInterval'    /etc/ssh/sshd_config || echo 'ClientAliveInterval 300'    >> /etc/ssh/sshd_config
grep -q '^ClientAliveCountMax'    /etc/ssh/sshd_config || echo 'ClientAliveCountMax 2'      >> /etc/ssh/sshd_config

Then reload the daemon to pick up the new config:

sudo systemctl reload ssh

The lab's systemctl is a stub — it prints what it would do but no real daemon is running. The file edit is what gets graded; sshd -T re-parses the config every invocation, so reload isn't actually required to verify here.
