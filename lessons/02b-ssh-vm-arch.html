Heads up — this is Arch Linux, not Ubuntu

CIS Ubuntu 24.04 is the actual benchmark target. v86 emulates 32-bit (i386) only, and modern Ubuntu/Debian dropped i386 — there's no pre-built Ubuntu image for v86 anywhere I can find. Arch Linux i686 is the closest "real distro" with a hosted snapshot, so this lab uses it instead. That's OK for this lesson: the audit and remediate commands (sudo sshd -T, sed -i, grep, systemctl reload ssh) are POSIX/Linux fundamentals — they behave identically on Arch. The only Arch-specific thing is pacman -S instead of apt install -y, and that doesn't come up in this lesson.

First boot pulls a ~15 MB state snapshot from i.copy.sh (BunnyCDN), then streams 9p filesystem chunks on demand. After that everything is browser-cached. If i.copy.sh is unreachable, fall back to the Buildroot lab.

Click ▶ Boot VM on the right (about 5 seconds — state snapshot resume, no kernel boot). The shell starts as root on tty1. The lab will overwrite /etc/ssh/sshd_config with a deliberately-weak config, then it's yours to harden.

🎯 Why this matters

SSH is the door to the box. The default OpenSSH config on most distros permits root login, password authentication, and unlimited idle sessions — every one of those is an attacker's best friend. This lesson tightens four CIS controls: disable root SSH login, disable password auth in favor of keys, drop MaxAuthTries from 6 to 4, and add an idle-session timeout.

📖 Tools used in this lesson

sshd — OpenSSH server daemon. sshd -T dumps the effective runtime config (file values merged with compile-time defaults). On this Arch lab it's the actual OpenSSH binary, not a stub. man 8 sshd · man 5 sshd_config

sed — stream editor. sed -i 's/old/new/' does in-place find-and-replace. man 1 sed

grep -q — quiet match, exit 0/1 — perfect for guards before echo … >> file. man 1 grep

systemctl reload — sends SIGHUP (or whatever the unit's ExecReload is) so the daemon re-reads its config without dropping connections. man 1 systemctl

🔍 Audit

Use sshd -T to see the effective running config — file values merged with OpenSSH's compile-time defaults. This is what the daemon would actually behave like at runtime, and what CIS auditors check:

sudo sshd -T | grep -iE 'permitrootlogin|passwordauth|maxauthtries|clientalive'

Or read the config file directly:

cat /etc/ssh/sshd_config

CIS-recommended targets:

  • PermitRootLogin = no
  • PasswordAuthentication = no
  • MaxAuthTries = 4
  • ClientAliveInterval = 300, ClientAliveCountMax = 2

🔧 Remediate

Apply all five directives at once with sed, then append any that weren't already present:

sed -i \
  -e 's/^#*PermitRootLogin .*/PermitRootLogin no/' \
  -e 's/^#*PasswordAuthentication .*/PasswordAuthentication no/' \
  -e 's/^#*MaxAuthTries .*/MaxAuthTries 4/' \
  -e 's/^#*ClientAliveInterval .*/ClientAliveInterval 300/' \
  -e 's/^#*ClientAliveCountMax .*/ClientAliveCountMax 2/' \
  /etc/ssh/sshd_config

grep -q '^PermitRootLogin'        /etc/ssh/sshd_config || echo 'PermitRootLogin no'        >> /etc/ssh/sshd_config
grep -q '^PasswordAuthentication' /etc/ssh/sshd_config || echo 'PasswordAuthentication no' >> /etc/ssh/sshd_config
grep -q '^MaxAuthTries'           /etc/ssh/sshd_config || echo 'MaxAuthTries 4'             >> /etc/ssh/sshd_config
grep -q '^ClientAliveInterval'    /etc/ssh/sshd_config || echo 'ClientAliveInterval 300'    >> /etc/ssh/sshd_config
grep -q '^ClientAliveCountMax'    /etc/ssh/sshd_config || echo 'ClientAliveCountMax 2'      >> /etc/ssh/sshd_config

Then reload the daemon (real systemd on this lab, not a stub):

systemctl reload sshd

On Arch the unit is named sshd.service (vs ssh.service on Debian/Ubuntu). sshd -T re-parses the config every invocation, so reload isn't strictly required to verify here.

✅ Verify

Re-run the audit. All five effective values should now match the CIS targets:

sudo sshd -T | grep -iE 'permitrootlogin|passwordauth|maxauthtries|clientalive'
# Expected:
permitrootlogin no
passwordauthentication no
maxauthtries 4
clientaliveinterval 300
clientalivecountmax 2
