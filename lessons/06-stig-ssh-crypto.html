Lesson 6 — STIG: Stricter SSH Crypto

DISA STIG~8 min3 controls

🎯 Why this matters

OpenSSH's defaults negotiate from a wide menu of ciphers, MACs, and key-exchange algorithms — including some that are old, weak, or not FIPS-approved. STIG locks the menu down to a FIPS 140-3-approved subset. Concretely: AES-CTR only (no CBC, no CHACHA20), HMAC-SHA2-256/512 only (no MD5 or SHA1 variants), modern ECDH or DH-group-exchange-SHA256 only.

This pairs with Lesson 2 — same file, same reload command. You're hardening the same surface, deeper.

📖 Tools and concepts used in this lesson

Cipher / MAC / KexAlgorithm — three sides of the SSH crypto triangle. Cipher encrypts the bulk traffic (AES-CTR). MAC (Message Authentication Code) authenticates each record (HMAC-SHA2). KexAlgorithm establishes the shared secret at session start (ECDH or DH-group-exchange-SHA256). Each is independently negotiated; weak choices in any of the three break the whole channel. OpenSSH protocol specs

FIPS 140-3 — NIST's standard for cryptographic modules used by US federal systems. Defines which algorithms and key sizes are "approved." STIG values are chosen to be the FIPS-approved subset of OpenSSH's algorithm catalog. FIPS 140-3 (NIST)

sshd -T — dumps OpenSSH's effective runtime config. Re-reads /etc/ssh/sshd_config every invocation, so it doesn't require a reload to verify changes. man 8 sshd · man 5 sshd_config

tee -a — read stdin and append to a file (-a) while also passing it through. sudo tee -a /etc/ssh/sshd_config >/dev/null is the standard idiom for "append to a root-owned file from a regular shell." man 1 tee

Heredoc <<'EOF' — single-quoted delimiter prevents shell expansion of $vars and backticks inside the body. Use whenever you're writing a config block that should ship verbatim. Bash heredoc reference

🔍 Audit

sudo sshd -T | grep -iE '^(ciphers|macs|kexalgorithms)'

Compare to STIG-required values:

  • ciphers aes256-ctr,aes192-ctr,aes128-ctr
  • macs hmac-sha2-512,hmac-sha2-256
  • kexalgorithms ecdh-sha2-nistp256,ecdh-sha2-nistp384,ecdh-sha2-nistp521,diffie-hellman-group16-sha512,diffie-hellman-group-exchange-sha256

🔧 Remediate

sudo tee -a /etc/ssh/sshd_config >/dev/null <<'EOF'

# DISA STIG — restrict SSH crypto to FIPS-approved subset
Ciphers aes256-ctr,aes192-ctr,aes128-ctr
MACs hmac-sha2-512,hmac-sha2-256
KexAlgorithms ecdh-sha2-nistp256,ecdh-sha2-nistp384,ecdh-sha2-nistp521,diffie-hellman-group16-sha512,diffie-hellman-group-exchange-sha256
EOF

sudo systemctl reload ssh

✅ Verify

sudo sshd -T | grep -iE '^(ciphers|macs|kexalgorithms)'
