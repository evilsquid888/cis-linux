Lesson 3a — UFW Host Firewall · with embedded Linux

CIS §3.5 ~10 min 4 controls ← read-only edition
Same content as Lesson 3, with a real Linux VM you can type into. The shell starts with ufw "installed" but inactive and default-allow — the bad starting state CIS wants you to fix. The stub ufw is stateful (rules persist across calls in /var/lib/ufw/state), so the audit/remediate/verify flow plays out exactly as it would on a real host.

🎯 Why this matters

A host firewall is your last line of defense if something else goes wrong — an exposed service binds 0.0.0.0 by accident, a config drifts, a teammate runs nc -lp 4444 for "just a minute." UFW (Uncomplicated Firewall) is Ubuntu's friendly nftables/iptables wrapper. CIS wants it installed, enabled, set to default-deny, with explicit allow rules only for what you actually need.

📖 Tools used in this lesson

ufw — Uncomplicated Firewall. A user-friendly front-end for managing nftables/iptables rules. Stores rules in /etc/ufw/user.rules (and v6 variants), kernel-loads them via iptables-restore or nft -f. The verbs you'll use: status (read), default (set policy), allow/deny (rules), enable/disable (toggle). man 8 ufw · Ubuntu UFW community docs

iptables / nftables — the kernel-level packet-filter engines that UFW programs. You can inspect what UFW actually loaded with iptables -L -n -v or nft list ruleset. Modern Ubuntu uses nftables under the hood by default. man 8 iptables · nftables wiki

ss — socket statistics. ss -tlnp lists TCP (-t) listening (-l) sockets with numeric addresses (-n) and the owning process (-p). Sanity-check before firewalling: what ports are *actually* exposed? man 8 ss

dpkg -l — Debian package list. dpkg -l ufw verifies the package is installed and shows its version. Status column starts with ii for "installed and ok". man 1 dpkg

apt — Debian/Ubuntu package manager front-end. apt install -y ufw installs without prompting. man 8 apt

Refresher: default-deny vs default-allow

Default-deny means "drop anything not explicitly allowed." Default-allow means "permit anything not explicitly denied." Default-deny is safer because new services don't accidentally become reachable.

🔍 Audit

Three quick checks: package installed, firewall active, and what's actually listening:

dpkg -l ufw | tail -1
sudo ufw status verbose

You're looking for: ufw installed, Status: active, default deny incoming, and only port 22 (SSH) allowed. The lab starts with the firewall inactive and default-allow — both findings.

🔧 Remediate

Order matters — allow SSH FIRST
If you enable default-deny before allowing SSH, you instantly lose your remote session on a real box. Run the ufw allow 22/tcp line first. 
sudo apt install -y ufw                # already installed in the lab
sudo ufw allow 22/tcp                  # <-- BEFORE the deny
sudo ufw default deny incoming
sudo ufw default allow outgoing
sudo ufw --force enable
sudo systemctl enable ufw

✅ Verify

sudo ufw status verbose
# Expected:
#   Status: active
#   Logging: on (low)
#   Default: deny (incoming), allow (outgoing), disabled (routed)
#   To              Action      From
#   --              ------      ----
#   22/tcp          ALLOW IN    Anywhere
