Lesson 6a — STIG: Stricter SSH Crypto · with embedded Linux

DISA STIG ~12 min 3 controls ← read-only edition
Same content as Lesson 6, with an in-browser Linux VM you can type into. The lab boots a deliberately-weak /etc/ssh/sshd_config (no Ciphers/MACs/KexAlgorithms set — so OpenSSH defaults apply, including SHA1-MAC and other STIG no-nos). The stub sshd mimics OpenSSH's full default crypto menu so sudo sshd -T shows realistic "before" output.

🎯 Why this matters

OpenSSH's defaults negotiate from a wide menu of ciphers, MACs, and key-exchange algorithms — including some that are old, weak, or not FIPS-approved. STIG locks the menu down to a FIPS 140-3-approved subset. Concretely: AES-CTR only (no CBC, no CHACHA20), HMAC-SHA2-256/512 only (no MD5 or SHA1 variants), modern ECDH or DH-group-exchange-SHA256 only.

📖 Tools and concepts used in this lesson

Cipher / MAC / KexAlgorithm — three sides of the SSH crypto triangle. Cipher encrypts the bulk traffic (AES-CTR). MAC authenticates each record (HMAC-SHA2). KexAlgorithm establishes the shared secret at session start (ECDH or DH-group-exchange-SHA256). Each is independently negotiated; weak choices in any of the three break the whole channel. OpenSSH protocol specs

FIPS 140-3 — NIST's standard for cryptographic modules. STIG values are the FIPS-approved subset of OpenSSH's algorithm catalog. FIPS 140-3 (NIST)

sshd -T — dumps the effective runtime config including OpenSSH's compiled-in algorithm defaults. man 5 sshd_config

tee -a — read stdin, append to a file, also pass through. sudo tee -a /etc/ssh/sshd_config >/dev/null <<'EOF' is the standard idiom for appending a config block to a root-owned file from a regular shell. man 1 tee

Heredoc <<'EOF' — single-quoted delimiter prevents shell expansion of $vars inside the body. Use whenever shipping a config block verbatim.

🔍 Audit

The lab starts with no explicit Ciphers/MACs/KexAlgorithms in /etc/ssh/sshd_config, so the stub sshd -T falls back to OpenSSH's full default menu — including SHA1-MAC and CHACHA20 cipher, both flagged by STIG.

sudo sshd -T | grep -iE '^(ciphers|macs|kexalgorithms)'

Compare to STIG-required values:

  • ciphers aes256-ctr,aes192-ctr,aes128-ctr
  • macs hmac-sha2-512,hmac-sha2-256
  • kexalgorithms ecdh-sha2-nistp256,ecdh-sha2-nistp384,ecdh-sha2-nistp521,diffie-hellman-group16-sha512,diffie-hellman-group-exchange-sha256

🔧 Remediate

Append the three STIG-mandated lines to /etc/ssh/sshd_config, then reload the daemon:

sudo tee -a /etc/ssh/sshd_config >/dev/null <<'EOF'

# DISA STIG — restrict SSH crypto to FIPS-approved subset
Ciphers aes256-ctr,aes192-ctr,aes128-ctr
MACs hmac-sha2-512,hmac-sha2-256
KexAlgorithms ecdh-sha2-nistp256,ecdh-sha2-nistp384,ecdh-sha2-nistp521,diffie-hellman-group16-sha512,diffie-hellman-group-exchange-sha256
EOF

sudo systemctl reload ssh

The lab's systemctl is a stub. sshd -T re-parses the config every invocation, so reload isn't required to verify here.

✅ Verify

Re-run the audit. The three lines should now show only the FIPS-approved values:

sudo sshd -T | grep -iE '^(ciphers|macs|kexalgorithms)'
# Expected:
ciphers aes256-ctr,aes192-ctr,aes128-ctr
kexalgorithms ecdh-sha2-nistp256,ecdh-sha2-nistp384,ecdh-sha2-nistp521,diffie-hellman-group16-sha512,diffie-hellman-group-exchange-sha256
macs hmac-sha2-512,hmac-sha2-256
