Lesson 7 — STIG: Password Complexity + Lockout

DISA STIG ~8 min 3 controls → try the in-browser Linux lab

🎯 Why this matters

Pairs with Lesson 4 (User Account Audit). STIG mandates stricter password rules and an explicit account-lockout policy via PAM. This is one of the clearest "STIG > CIS defaults" examples — same files, just stricter values.

Two configs touch this:

  • /etc/security/pwquality.conf — what counts as a "good enough" new password
  • /etc/pam.d/common-auth + common-account — what happens after N failed logins (faillock)
📖 Tools and concepts used in this lesson

PAM (Pluggable Authentication Modules) — the framework Linux uses to decide "is this login allowed?" Modules are stacked in /etc/pam.d/<service> files. Each line is type (auth/account/password/session) + control (required/sufficient/[default=die]/etc.) + module + arguments. The order matters — auth modules are evaluated top to bottom. man 7 PAM · man 5 pam.conf

pam_pwquality.so — checks proposed new passwords against a quality policy (length, character classes, dictionary words). Configured via /etc/security/pwquality.conf. The credit values (ucredit/lcredit/dcredit/ocredit) work both ways: positive = "you get N free chars off minlen for using this class," negative = "you must include at least N of this class." STIG uses -1 across the board for "require at least one of each." man 8 pam_pwquality · libpwquality on GitHub

pam_faillock.so — tracks consecutive authentication failures per user and locks the account when a threshold is hit. Replaces the older pam_tally2. Two installation points: preauth (check current count before letting auth proceed), authfail (increment count on failure). man 8 pam_faillock · man 8 faillock (the unlock CLI)

libpam-pwquality — the Debian/Ubuntu package that ships pam_pwquality.so. pam_faillock ships in libpam-modules (already installed on Ubuntu 24.04). libpam-pwquality on Ubuntu

sed -i '/pattern/i ...' — insert a line BEFORE every line matching pattern. '/pattern/a ...' appends after. The remediate uses both to slot pam_faillock entries on either side of the existing pam_unix line. GNU sed: insert/append commands

🔍 Audit

# Password rules
grep -E '^(minlen|ucredit|lcredit|dcredit|ocredit)\s' /etc/security/pwquality.conf

# Lockout (PAM)
grep -E 'pam_faillock' /etc/pam.d/common-auth /etc/pam.d/common-account

STIG-required values:

  • minlen = 15
  • ucredit = -1 (≥1 uppercase required)
  • lcredit = -1 (≥1 lowercase required)
  • dcredit = -1 (≥1 digit required)
  • ocredit = -1 (≥1 special char required)
  • pam_faillock with deny=3 unlock_time=900 in both common-auth and common-account

🔧 Remediate

sudo apt install -y libpam-pwquality

sudo tee -a /etc/security/pwquality.conf >/dev/null <<'EOF'

# DISA STIG password complexity
minlen = 15
ucredit = -1
lcredit = -1
dcredit = -1
ocredit = -1
EOF

# Faillock — Ubuntu 24.04 ships pam_faillock; enable via pam-auth-update or directly
sudo sed -i '/pam_unix.so/i auth required pam_faillock.so preauth deny=3 unlock_time=900' /etc/pam.d/common-auth
sudo sed -i '/pam_unix.so/a auth [default=die] pam_faillock.so authfail deny=3 unlock_time=900' /etc/pam.d/common-auth
sudo sed -i '/pam_unix.so/i account required pam_faillock.so' /etc/pam.d/common-account
Test in a second session
PAM mistakes can lock you out of all logins. Keep a privileged shell open while you test, and try logging in fresh from another terminal before moving on.

✅ Verify

grep -E '^(minlen|ucredit|lcredit|dcredit|ocredit)\s' /etc/security/pwquality.conf
