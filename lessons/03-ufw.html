Lesson 3 — UFW Host Firewall

CIS §3.5 ~7 min 4 controls → try the in-browser Linux lab

🎯 Why this matters

A host firewall is your last line of defense if something else goes wrong — an exposed service binds 0.0.0.0 by accident, a config drifts, a teammate runs nc -lp 4444 for "just a minute." UFW (Uncomplicated Firewall) is Ubuntu's friendly nftables/iptables wrapper. CIS wants it installed, enabled, set to default-deny, with explicit allow rules only for what you actually need.

Refresher: default-deny vs default-allow

Default-deny means "drop anything not explicitly allowed." Default-allow means "permit anything not explicitly denied." Default-deny is safer because new services don't accidentally become reachable.

📖 Tools used in this lesson

ufw — Uncomplicated Firewall, Ubuntu's user-friendly front-end for nftables/iptables. Verbs you'll use: status, default, allow/deny, enable/disable. man 8 ufw · Ubuntu UFW community docs

iptables / nftables — kernel-level packet filters that UFW programs. Inspect what UFW actually loaded with iptables -L -n -v or nft list ruleset. Modern Ubuntu uses nftables under the hood. man 8 iptables · nftables wiki

ss — socket statistics. ss -tlnp = TCP, listening, numeric, with process. Sanity-check before firewalling: what's actually exposed? man 8 ss

dpkg -l — Debian package list. The ii in column 1–2 means "installed, ok". man 1 dpkg

apt — Debian/Ubuntu package manager. man 8 apt

🔍 Audit

dpkg -l ufw | tail -1
sudo ufw status verbose
ss -tlnp        # what's listening, totally separate from ufw — sanity check

You're looking for: ufw installed, Status: active, default deny incoming, only port 22 (SSH) allowed.

🔧 Remediate

Order matters — allow SSH FIRST
If you enable default-deny before allowing SSH, you instantly lose your remote session. Run the ufw allow 22/tcp line first. 
sudo apt install -y ufw
sudo ufw allow 22/tcp                  # <-- BEFORE the deny
sudo ufw default deny incoming
sudo ufw default allow outgoing
sudo ufw --force enable
sudo systemctl enable ufw

✅ Verify

sudo ufw status verbose
# Expected:
#   Status: active
#   Logging: on (low)
#   Default: deny (incoming), allow (outgoing), disabled (routed)
#   To              Action      From
#   --              ------      ----
#   22/tcp          ALLOW IN    Anywhere
